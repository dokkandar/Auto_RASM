pline: tapered width (model + render + tool) — stages 1-3

Adds true per-segment tapered width to polylines.

Model (cad_kernel):
- Polyline gains `widths: Vec<(f64,f64)>` (per-segment start/end width; empty =
  thin = current behaviour). All ~24 Polyline literals updated.
- Transforms preserve widths; scale multiplies them; reversed() reverses +
  swaps each (start,end). bbox() expands by max half-width. 170 tests pass.

Render (cad_app):
- New draw_polyline_widths: each segment's centerline is tessellated (arcs
  become real arcs), width lerps start->end, and each sub-span fills as a
  convex quad offset +/- half-width along the local normal. Widths are world
  units, so the fill scales with zoom. Solid polylines route through this;
  dashed-linetype + selection highlight still stroke the centerline (v1).

Tool (cad_app):
- `w`/`h` (width/halfwidth) sub-command: enter starting then ending width
  (Enter on ending = same; halfwidth doubles). The end carries as the next
  segment's start (AutoCAD behaviour). Stored per-segment in pending_widths,
  drained into Polyline.widths on commit (Close/Enter/auto-close).
- All pline reset paths (Esc/cancel/undo) keep widths in sync with bulges.

NOT yet: DXF/RSM persistence of widths (stage 4) — save/reload currently drops
width (read paths default to empty). Coming next.
